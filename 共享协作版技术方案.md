# UCS展车外借推荐系统 — 共享协作版技术方案

## 一、需求升级概述

将原有的单机 HTML 工具升级为**带权限管理和审批流程的在线协作系统**：

| 角色 | 权限 |
|------|------|
| **管理员（你）** | 上传/更新展车数据、上传/更新门店数据、审批销售的展车申请 |
| **销售（Fellow）** | 检索展车、提交展车借用申请、查看申请状态 |

## 五、核心业务流程

### 5.1 销售检索展车
```
销售打开网页 → 输入需求（如"城北万象城要一辆es8，白色"）
    ↓
系统模糊匹配车型+门店 → 按规则排序（CDC优先→同区优先→配置匹配优先）
    ↓
展示推荐列表（含实际到达点位、车漆、内饰等信息）
    ↓
销售点击「申请借用」→ 填写姓名 → 提交申请
    ↓
申请状态变为「待审批」，该车锁定（其他人不可再申请）
```

### 5.2 管理员审批流程
```
管理员打开管理后台 → 查看「待审批」列表
    ↓
收到新申请通知（页面实时提示 / 声音提醒）
    ↓
查看申请详情（销售、目标门店、车辆信息）
    ↓
点击「同意」→ 车辆标记为已选用（从可借池移除）→ 销售收到通过通知
    ↓
或点击「否决」→ 车辆恢复可借状态 → 销售收到拒绝通知（附原因）
```

### 5.3 管理员更新数据
```
管理员登录后台 → 进入「数据管理」页面
    ↓
上传新的展车列表 Excel → 系统解析并全量替换 cars 表
    ↓
上传新的门店信息 Excel → 系统解析并全量替换 stores 表
    ↓
可选择「重置已选用状态」（清空 isDeleted 标记，恢复全部车辆）
```

## 六、页面结构设计

### 6.1 销售端页面 (`/` 或 `/sales`)
- **检索区**：需求输入框 + 车漆/内饰偏好 + 搜索按钮
- **结果区**：优先推荐 / 备选推荐卡片列表
- **申请表单**：点击「申请」后弹窗填写姓名提交
- **我的申请**：查看自己提交的所有申请及状态

### 6.2 管理端页面 (`/admin`)
- **登录页**：用户名密码登录
- **数据管理**：上传展车Excel + 上传门店Excel + 数据预览
- **审批中心**：待审批列表（实时刷新）+ 已审批历史
- **系统设置**：重置车辆状态、管理销售账号

## 七、技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | **Next.js 14** (App Router) | React全栈框架，一套代码搞定前后端 |
| UI组件 | **Tailwind CSS + shadcn/ui** | 美观、现代、响应式 |
| 后端API | **Next.js API Routes** | 内置API路由，无需单独后端 |
| 数据库 | **MongoDB Atlas** | 免费512MB额度，足够使用 |
| ORM | **Mongoose** | MongoDB的Node.js驱动 |
| 认证 | **NextAuth.js** | 简单安全的认证方案 |
| 部署 | **Vercel** | 一键部署，自动CI/CD |
| 文件存储 | **Vercel 或内存解析** | Excel直接解析后入库，无需持久存储 |

## 十、安全性说明

- **展车数据**：存储在 MongoDB 中，只有管理员能通过后台上传更新
- **销售端**：无需登录即可检索，但提交申请时可要求填写姓名
- **管理端**：需要密码登录，建议设置强密码
- **数据传输**：Vercel 自动提供 HTTPS 加密
