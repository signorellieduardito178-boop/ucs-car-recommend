# UCS展车系统 — 轻量版技术方案（静态网站 + 飞书多维表）

## 一、需求变更概述

从「全栈应用（Next.js + MongoDB + Vercel）」降级为「轻量静态网站 + 飞书多维表审批」，解决国内访问慢、服务器依赖重的问题。

| 维度 | 原方案 | 新方案 |
|------|--------|--------|
| 部署平台 | Vercel（美国节点） | Vercel / GitHub Pages / Cloudflare Pages |
| 数据库 | MongoDB Atlas（需要网络连通） | 无数据库，纯 JSON 文件 |
| 后端服务 | Next.js API Routes（需要服务器运行） | 无后端，纯静态页面 |
| 审批流程 | 网页内审批（管理员点同意/否决） | 飞书多维表审批（飞书自带消息推送） |
| 数据更新 | 网页上传 Excel → 写入 MongoDB | 本地脚本转换 Excel → JSON → Git Push → 自动部署 |
| 销售申请 | 网页内点击「申请」→ 提交到后台 | 网页选中车辆 → 跳转飞书多维表 → 填写表单 |

## 三、核心数据流

### 3.1 数据更新流程（管理员每日操作）

```
1. 你从系统导出最新的展车列表 Excel
2. 运行本地脚本（或手动）：Excel → 转换为 JSON
3. 将 JSON 文件提交到 GitHub：git add . && git commit && git push
4. Vercel/Cloudflare 检测到代码变更，自动重新构建部署（约 1-2 分钟）
5. 销售刷新网页，即可看到最新的展车数据
```

### 3.2 销售查询流程

```
1. 销售打开网站（手机/电脑均可）
2. 填写姓名、所在门店
3. 输入需求（如：城北万象城要一辆es8，白色）
4. 点击「获取推荐」，网页从静态 JSON 中实时筛选
5. 查看推荐结果（优先/备选），找到合适的车辆
6. 点击「去飞书申请」→ 跳转到飞书多维表申请表单
7. 飞书表单已预填好车架号、车型、车漆等信息，销售补充信息后提交
8. 提交后飞书自动推送审批消息给你（你设置在飞书多维表中的审批流程）
```

### 3.3 你的审批流程（在飞书内完成）

```
1. 你收到飞书审批消息（手机/电脑）
2. 打开飞书多维表，查看申请详情
3. 点击「通过」或「驳回」
4. 如果通过，你手动在 Excel 中标记该车辆为已选用
5. 次日更新 Excel 并重新部署网站（车辆自动从可借池移除）
```

## 四、技术选型对比

| 托管平台 | 国内访问速度 | 自动部署 | 费用 | 推荐度 |
|----------|-------------|---------|------|--------|
| **Vercel** | ⭐⭐ 较慢 | ✅ 自动 | 免费 | 保留现状（已可用） |
| **GitHub Pages** | ⭐⭐ 较慢 | ✅ 自动 | 免费 | 与 Vercel 速度相近 |
| **Cloudflare Pages** | ⭐⭐⭐⭐ 较快 | ✅ 自动 | 免费 | ⭐ 推荐切换 |
| **腾讯云轻量服务器** | ⭐⭐⭐⭐⭐ 最快 | ❌ 手动 | ~50元/月 | 长期最佳 |

**推荐方案**：
- **短期**：继续用 Vercel（网站已能跑，只是慢点），等调试稳定后再迁移
- **中期**：切换到 **Cloudflare Pages**（免费 + 国内访问快 + 自动部署）
- **长期**：腾讯云轻量服务器（完全可控 + 可绑定备案域名）

## 五、飞书多维表集成方案

### 5.1 方案 A：URL 参数预填充（推荐）

飞书多维表的表单视图支持通过 URL 参数预填充字段，格式如下：

```
https://base.feishu.cn/base/你的BaseID/form/你的表单ID
  ?prefill_VIN=LSxxxxxx
  &prefill_车型=ES8
  &prefill_车漆=云白
  &prefill_目标门店=城北万象城
  &prefill_销售姓名=张三
  &prefill_所在门店=钱江新城
```

**实现方式**：
- 销售点击「去飞书申请」时，网页拼接上述 URL
- 打开飞书表单后，字段已经自动填好，销售只需补充其他信息（如用途、日期等）
- 提交后飞书自动触发审批消息推送

### 5.2 方案 B：复制信息 + 手动粘贴（保底）

如果飞书多维表不支持某些字段的 URL 预填充：
- 销售点击「复制车辆信息」
- 网页弹出一个格式化的文本（车架号、车型、车漆、门店等）
- 销售点击跳转到飞书多维表
- 手动粘贴信息到对应字段

### 5.3 方案 C：飞书机器人推送（辅助）

通过飞书 webhook，销售点击申请后，网页后端发送一条消息到飞书群：
```
🚗 新的展车申请
销售: 张三 (钱江新城)
车型: ES8 | 车架号: LSxxxxxx
车漆: 云白 | 目标门店: 城北万象城
请在多维表中审批: [链接]
```

**注意**：由于新方案是无后端的静态网站，飞书推送功能需要改为：
- 销售选中车辆后，网页调用一个免费的 Serverless 函数（如 Vercel Edge Function 或 Cloudflare Worker）
- 或者放弃自动推送，依靠飞书多维表自身的审批通知

## 八、优缺点总结

### ✅ 优点
1. **无需服务器**：纯静态网站，不依赖任何数据库和后端服务
2. **国内可访问**：Vercel 虽然慢但可以访问，切换到 Cloudflare Pages 更快
3. **数据完全可控**：数据在 JSON 文件中，你随时更新
4. **审批流程完善**：飞书多维表自带审批、消息推送、权限管理
5. **成本极低**：几乎零成本（只需要 GitHub + 免费托管平台）

### ⚠️ 缺点/注意点
1. **数据更新有延迟**：Git push 后需要 1-2 分钟重新部署，不是秒级实时
2. **飞书跳转体验**：需要销售手动在飞书中补充一些信息（但车辆信息已预填）
3. **已选用车辆状态**：需要你手动维护 Excel 中标记，次日更新到网站
4. **并发申请风险**：两个销售同时申请同一辆车（飞书端需要配置校验规则）
